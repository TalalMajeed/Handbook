# Handbook QA System

A scalable Question-Answering system over the **NUST SEECS UG & PG Handbooks**, built using Big Data techniques including Locality Sensitive Hashing (LSH), MinHash, SimHash, TF-IDF, PageRank, and neural sentence embeddings.

---

## System Overview

```
PDF Handbooks
    │
    ▼
[1] Data Ingestion        — PyMuPDF, word-window chunking (60–300 words, 50% overlap)
    │
    ▼
[2] SimHash Dedup         — 64-bit SimHash fingerprints + Hamming distance threshold
    │
    ▼
[3] Sentence Embeddings   — all-MiniLM-L6-v2 (384-dim, normalized)
    │
    ├──▶ [4a] PageRank     — cosine-similarity graph → networkx.pagerank
    │
    ├──▶ [4b] TF-IDF       — sklearn TfidfVectorizer (20k features)
    │
    └──▶ [4c] MinHash+LSH  — 128 hash functions, 16 bands, s*≈0.707
         │
         ▼
[5] Query Processing
    ├── retrieve_tfidf()   → exact baseline
    ├── retrieve_lsh()     → approximate (LSH buckets → Jaccard re-rank)
    └── retrieve()         → hybrid (TF-IDF pre-filter + semantic + PageRank)
         │
         ▼
[6] Answer Generation     — google/flan-t5-base (seq2seq, local, streamed)
         │
         ▼
[7] Web Interface         — FastAPI + SSE streaming + evidence panel (HTML/CSS/JS)
```

## Usage

### Step 1 — Build the index (run once)

```bash
python build_index.py
```

This reads both PDFs, chunks them, computes embeddings, builds PageRank, TF-IDF, and MinHash+LSH, then saves all artifacts to `handbook-1.0/`. Takes ~1–2 minutes on CPU.

### Step 2 — Start the server

```bash
python server.py
```

Open `http://localhost:8000` in your browser.

### Step 3 (optional) — Run experiments

```bash
python experiments.py
```

Prints a comparison table of all three retrieval methods across 15 sample queries.

## Core Algorithms

### MinHash + LSH

Each chunk is represented as a set of word 3-shingles. A MinHash signature of length 128 is computed using the universal hash family `h(x) = (a·x + b) mod p` (Mersenne prime). Signatures are divided into 16 bands of 8 rows each for LSH banding. The theoretical Jaccard threshold at which two documents collide with probability 0.5 is `s* = (1/16)^(1/8) ≈ 0.707`.

### SimHash

Each chunk's 64-bit SimHash fingerprint is computed by bit-projecting token MD5 hashes into a weighted accumulator vector. Near-duplicates (Hamming distance ≤ 3) are removed before indexing.

### TF-IDF Baseline

`sklearn.TfidfVectorizer` with 20,000 features. Cosine similarity is computed as a sparse matrix–vector product at query time. Exact, O(|vocab| × |chunks|).

### Hybrid Retrieval

```
score = 0.60 × TF-IDF_cosine + 0.25 × Semantic_cosine + 0.15 × PageRank
```

TF-IDF pre-filters to top-30 candidates; semantic re-ranking uses the sentence-transformer embedding. PageRank weights chunks by their centrality in the chunk-similarity graph.
