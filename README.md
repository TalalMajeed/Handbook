# Handbook QA System

A scalable Question-Answering system over the **NUST SEECS UG & PG Handbooks**, built using Big Data techniques including Locality Sensitive Hashing (LSH), MinHash, SimHash, TF-IDF, PageRank, and neural sentence embeddings.

---

## System Overview

```
PDF Handbooks
    │
    ▼
[1] Data Ingestion        — PyMuPDF, header stripping, sentence-aware chunking
                            (50–250 words, 1-sentence overlap)
    │
    ▼
[2] SimHash Dedup         — 64-bit SimHash fingerprints + Hamming distance ≤ 3
    │
    ▼
[3] Sentence Embeddings   — all-MiniLM-L6-v2 (384-dim, normalized)
    │
    ├──▶ [4a] PageRank     — cosine-similarity graph → networkx.pagerank
    │
    ├──▶ [4b] TF-IDF       — sklearn TfidfVectorizer (15k features, bigrams,
    │                         sublinear TF, domain stop-words, synonym expansion)
    │
    └──▶ [4c] MinHash+LSH  — 128 hash functions, 16 bands, s*≈0.707
         │
         ▼
[5] Query Processing
    ├── retrieve_tfidf()   → exact baseline (synonym-expanded query)
    ├── retrieve_lsh()     → approximate (LSH buckets → Jaccard re-rank)
    └── retrieve()         → hybrid (TF-IDF gate + semantic + PageRank rerank)
         │
         ▼
[6] Answer Generation     — Groq API: llama-3.1-8b-instant (free, streaming)
                            Fallback: flan-t5-base (local) → extractive
         │
         ▼
[7] Web Interface         — FastAPI + SSE streaming + evidence panel (HTML/CSS/JS)
```

## Setup — Groq API Key (free, required for best answers)

The system uses the **Groq API** for answer generation. Groq is completely free — no credit card needed.

1. Go to **https://console.groq.com/** and sign up with Google or GitHub
2. Click **API Keys → Create API Key**, copy the key (starts with `gsk_...`)
3. Paste it into the `.env` file in the project root:

```
GROQ_API_KEY=gsk_your_key_here
```

That's it. The server reads the key automatically on startup — no terminal export needed.

> **No key?** The server falls back to local `flan-t5-base` generation, then to
> pure extractive answers. The system still works, but answer quality is lower.

## Usage

### Step 1 — Build the index (run once, or after updating PDFs)

```bash
python build_index.py
```

Reads both PDFs, strips headers, chunks by sentence boundaries, computes embeddings,
builds PageRank, TF-IDF (with synonym expansion), and MinHash+LSH. Saves all
artifacts to `handbook-1.0/`. Takes ~2–3 minutes on CPU.

### Step 2 — Start the server

```bash
python server.py
```

Open **http://localhost:8000** in your browser.

### Step 3 (optional) — Run experiments

```bash
python experiments.py
```

Prints a full comparison table of all three retrieval methods across 15 sample
queries: latency, Precision@8, and scalability results. Required for the report.

## Core Algorithms

### MinHash + LSH

Each chunk is represented as a set of word 3-shingles. A MinHash signature of
length 128 is computed using the universal hash family `h(x) = (a·x + b) mod p`
(Mersenne prime). Signatures are divided into 16 bands of 8 rows each for LSH
banding. The theoretical Jaccard threshold at which two documents collide with
probability 0.5 is `s* = (1/16)^(1/8) ≈ 0.707`.

If no LSH bucket matches (short query → few shingles → low Jaccard with long
chunks), the system falls back to a linear MinHash Jaccard scan over all chunks.
The method tag `lsh-fallback` identifies this path in experiments.

### SimHash

Each chunk's 64-bit SimHash fingerprint is computed by bit-projecting token MD5
hashes into a weighted accumulator vector. Near-duplicates with Hamming distance
≤ 3 are removed before indexing.

### TF-IDF Baseline

`sklearn.TfidfVectorizer` with 15,000 features, bigram support `(1,2)`,
`sublinear_tf=True`, `max_df=0.85`, and domain stop-words (`nust`, `university`,
`student`, `handbook`, etc.). Cosine similarity is computed as a sparse
matrix–vector product at query time — exact, O(|vocab| × |chunks|).

**Synonym expansion** is applied at both index time and query time so that
vocabulary mismatches like `GPA` vs `CGPA`, or `UG` vs `undergraduate`, are
resolved before scoring.

### Hybrid Retrieval

```
score = 0.70 × TF-IDF_cosine + 0.28 × Semantic_cosine + 0.02 × PageRank
```

- TF-IDF pre-filters to top-40 candidates after applying a `MIN_TFIDF_SCORE=0.005`
  gate — chunks with zero vocabulary overlap are excluded entirely, preventing
  high-PageRank but irrelevant chunks from surfacing.
- Semantic re-ranking uses the sentence-transformer embedding on the cleaned
  (unexpanded) query to avoid synonym noise in the vector space.
- PageRank is a tiebreaker only (weight reduced from 0.15 → 0.02 after it was
  found to boost popular but irrelevant chunks to the top).

### Answer Generation

Answers are generated by **Groq API** (`llama-3.1-8b-instant`) reading only the
top-5 retrieved chunks. Context is passed without inline source tags to prevent
tag leakage into answers. Citations are appended by Python after generation.

Fallback chain if Groq is unavailable:
1. Local `flan-t5-base` (seq2seq, deterministic, `do_sample=False`)
2. Pure extractive: best-matching sentences copied directly from retrieved chunks

## Key Fixes Applied During Development

| Problem | Root Cause | Fix |
|---|---|---|
| Irrelevant chunks at top | PageRank weight 0.15 boosted popular but off-topic chunks | Reduced to 0.02; added MIN_TFIDF_SCORE gate |
| "GPA" query missed "CGPA" chunks | TF-IDF has no semantic generalisation | Synonym expansion at index + query time |
| Headers polluting every chunk | PyMuPDF embeds running headers in page text | `strip_headers()` removes them before chunking |
| Rules split across chunks | Blind word-window chunker ignored sentences | Sentence-aware chunker keeps rules intact |
| One-word / truncated answers | `flan-t5-base` (250M params) cannot do reading comprehension or tables | Replaced with Groq API (Llama 3.1 8B) |
| Source tags in answers | `[ug-handbook, p.25]` tags inside context string were echoed | Tags stripped from context; citations added by Python |
| "Not found" echoed by model | Fallback phrase was inside the prompt instruction | Fallback handled in Python, not in the model prompt |
| Dots stripped from query | `_clean()` removed `.` so `2.75` became `275` | Preserved `.`, `-`, `%` in cleaner regex |
